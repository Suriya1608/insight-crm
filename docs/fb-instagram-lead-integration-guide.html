Overview & Architecture

What was built

Two separate integrations were implemented inside the CRM on the feature/fb-insta-integrate branch:

  • Facebook / Instagram Lead Ads — When a user fills a lead form on Facebook or Instagram, Meta sends a real-time webhook to the CRM. The CRM fetches the lead details from the Graph API, creates a lead record, auto-assigns it to a manager, and sends a notification.
  • Instagram Direct Messages (DM) — Inbound Instagram DMs are received via a separate webhook and stored as conversations. CRM users can read and reply from within the CRM UI.

Part 1 – Create & Configure a Meta App

ℹ️
Do this once per client. You will need a Facebook Business Manager account and admin access to the client's Facebook Page.
1

Go to Meta for Developers

Open developers.facebook.com → click My Apps → Create App.

2

Select App Type

Choose "Business" as the app type. This unlocks the Lead Ads and Instagram products.

3

Add Products

From the app dashboard, click Add Product and add both:

  • Facebook Login (needed to generate page tokens)
  • Webhooks
  • Instagram (if using IG DMs)
4

Note Your App Credentials

Go to App Settings → Basic. Copy and save:

  • App ID (e.g. 1234567890123456)
  • App Secret (click "Show" to reveal)
5

Generate a Long-Lived Page Access Token

Use the Graph API Explorer (developers.facebook.com/tools/explorer):

  1. Select your app and the client's page
  2. Request permissions: leads_retrieval, pages_manage_metadata, pages_read_engagement
  3. Get a short-lived User Token, then exchange it for a long-lived Page Token via:
GET https://graph.facebook.com/v19.0/{PAGE_ID}
    ?fields=access_token
    &access_token={USER_LONG_LIVED_TOKEN}
⚠️
Page Access Tokens do not expire as long as they remain active. If the user removes the app from the page, the token is invalidated.

Part 2 – Facebook Lead Ads Webhook Setup

1

Routes must be publicly reachable

The CRM server must be accessible from the internet. The webhook URLs are:

# Verification (GET) + Lead events (POST)
GET  /webhooks/meta/facebook
POST /webhooks/meta/facebook

# Instagram DM webhook
GET  /webhooks/meta/instagram
POST /webhooks/meta/instagram
⚠️
These routes are excluded from CSRF middleware in app/Http/Middleware/VerifyCsrfToken.php. Make sure that is already set up:
// VerifyCsrfToken.php
protected $except = [
    'webhooks/meta/*',
];
2

Create a Verify Token

Make up any random secret string, e.g. my_client_verify_abc123. You will enter this in both the Meta dashboard and the CRM settings.

3

Configure Webhook in Meta Dashboard

In your Meta App, go to Webhooks → click Subscribe to this object for Page:

  • Callback URL: https://your-crm-domain.com/webhooks/meta/facebook
  • Verify Token: the string you created above

Click Verify and Save. Meta will send a GET request to your CRM with hub_mode=subscribe — the CRM responds with the challenge to confirm.

4

Subscribe to the "leadgen" field

After the webhook is verified, click Edit on the Page object and check the leadgen field. Save.

ℹ️
The CRM also has a one-click "Subscribe App to Page" button in the Facebook settings page that does this via Graph API automatically.

How Verification Works (Code Reference)

// SocialMediaController::handleVerification()
$mode      = $request->query('hub_mode');         // must be 'subscribe'
$token     = $request->query('hub_verify_token'); // must match saved token
$challenge = $request->query('hub_challenge');    // echo this back

if ($mode === 'subscribe' && hash_equals($savedToken, $token)) {
    return response($challenge, 200);  // ✅ verified
}
return response('Forbidden', 403);    // ❌ rejected

How HMAC Signature Validation Works

Every POST from Meta includes a header X-Hub-Signature-256: sha256=<hash>. The CRM validates it using your App Secret:

$appSecret = Setting::getSecure('fb_leads_app_secret');
$expected  = 'sha256=' . hash_hmac('sha256', $request->getContent(), $appSecret);

if (! hash_equals($expected, $sig)) {
    return response('Forbidden', 403);  // reject tampered payloads
}

Part 4 – Subscribe App to Facebook Page

1

Click "Subscribe App to Page" in CRM

After saving settings, click the Subscribe App to Page button on the Facebook settings page. This calls:

POST https://graph.facebook.com/v19.0/{PAGE_ID}/subscribed_apps
Body: { subscribed_fields: "leadgen", access_token: "{PAGE_TOKEN}" }

On success you will see: "App successfully subscribed to the page. Leadgen webhook is active."

2

Verify Subscription in Meta Dashboard

In your Meta App → Webhooks → Page object, you should now see the page listed as subscribed under leadgen.

⚠️
Common issue: If the subscription fails with a permission error, the Page Access Token may not have the pages_manage_metadata permission. Re-generate the token with that permission from the Graph API Explorer.

Instagram vs Facebook Source Detection

When a lead comes through, the CRM checks whether the ad ran on Instagram or Facebook by querying the ad object:

GET https://graph.facebook.com/v19.0/{AD_ID}
    ?fields=publisher_platforms
    &access_token={PAGE_TOKEN}

// Response: { "publisher_platforms": ["instagram"] }
// → source = 'instagram_ads'

// Response: { "publisher_platforms": ["facebook"] }
// → source = 'facebook_ads'

Part 6 – Instagram DM Integration

ℹ️
Instagram DM requires the Facebook Page to be connected to an Instagram Business/Creator account. Personal Instagram accounts are not supported by the API.
1

Connect Instagram Business Account

In Facebook Page Settings → Instagram, connect the client's Instagram Business account. This is required before any API access.

2

Get Instagram User ID

GET https://graph.facebook.com/v19.0/{PAGE_ID}
    ?fields=instagram_business_account
    &access_token={PAGE_TOKEN}

// Response: { "instagram_business_account": { "id": "17841400123456789" } }

Copy the id — this is the Instagram User ID you need.

3

Configure Instagram Webhook in Meta App

In your Meta App → Webhooks → subscribe to the Instagram object:

  • Callback URL: https://your-crm-domain.com/webhooks/meta/instagram
  • Verify Token: your custom verify token

Subscribe to the messages field.

4

Add Instagram Account in CRM

Go to Admin → Settings → Instagram. Click Add Account and fill in:

FieldValue
NameDisplay name for this account (e.g. client brand name)
Page IDFacebook Page ID
Instagram User IDFrom Step 2 above
Access TokenLong-lived Page Access Token (encrypted at rest)
App SecretOptional — for webhook signature validation
Verify TokenSame token used in Meta Webhooks dashboard
5

Test DM Flow

Send a DM to the Instagram account from a personal Instagram. Within seconds it should appear in Manager → Instagram in the CRM. The CRM stores:

  • Sender's PSID (Page-Scoped User ID) and username
  • Message content and timestamp
  • Unread count per conversation
6

Reply from CRM

Managers can reply directly in the CRM. Replies are sent via:

POST https://graph.facebook.com/v19.0/{IG_USER_ID}/messages
Body: {
    recipient: { id: "{SENDER_PSID}" },
    message:   { text: "Your reply here" },
    access_token: "{PAGE_TOKEN}"
}

Troubleshooting

Webhook verification fails (403)

❌
Symptom: Meta dashboard shows "Callback URL failed" during verification.

Causes & fixes:

  • The Verify Token in Meta dashboard does not match the one saved in CRM settings → update to match exactly.
  • The CRM URL is not publicly reachable (local dev server) → use ngrok or deploy to server first.
  • CSRF protection is blocking the GET request → ensure webhooks/meta/* is in VerifyCsrfToken::$except.

Webhook verified but leads not appearing

  • App is not subscribed to the page → click "Subscribe App to Page" in CRM settings.
  • Page Access Token expired or lacks leads_retrieval permission → regenerate the token.
  • HMAC mismatch → check that App Secret in CRM matches the one in Meta App Settings.
  • Check storage/logs/laravel.log for detailed error messages.

Page subscription fails ("Subscribe App to Page" button)

  • Token must have pages_manage_metadata permission.
  • The app must be in Live mode (not Development mode) or the page admin must be a test user.
  • The Page ID saved in CRM must exactly match the page you are subscribing.

Lead source always shows "facebook_ads" even for Instagram ads

The CRM checks publisher_platforms via the Graph API. If the Ad ID is not accessible with the current token, it defaults to facebook_ads. Ensure the page token has ad read permissions, or this is acceptable behavior since the lead data is the same either way.

Duplicate leads appearing

The CRM has a 10-minute dedup window by phone number. If the same lead is submitted twice within 10 minutes it is skipped. For longer dedup windows, update this check in SocialMediaController::createLeadFromFacebook():

Lead::where('phone', $phone)
    ->where('created_at', '>=', now()->subMinutes(10))  // ← change this value
    ->exists();

Instagram DM not appearing in CRM

  • The Instagram account must be a Business or Creator account, not personal.
  • Webhook must be subscribed to the messages field on the Instagram object (not Page object).
  • Only one Instagram account can be active at a time in CRM settings.
  • Check that the Instagram User ID is correct (not the Page ID).
